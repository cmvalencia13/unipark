Implementation Order — UI-First, 6 Phases

Each phase produces reviewable code. Fake data first, real integration after.

1

Project Skeleton + Design System

Gradle setup (Kotlin DSL, Version Catalogs, Hilt, Compose, dependencies). UniParkTheme (Material3 dark + Stitch colors). Typography (Sora + Inter). GlassPanel, LiquidGlassPanel, GlowActivePanel, StatusPill, OccupancyBar, ShineButton, StatusDot, PulseRing modifiers. Empty MainActivity with TopAppBar + BottomNavBar shell.

2

Dashboard Screen (Status Tab)

ActivePermitCard (hero, glow-active), CurrentLocationCard, CampusTrendsCard (OccupancyBar x2), SystemAlertsCard. Staggered entrance animations. All with fake ViewModel data. Tab 1 of bottom nav functional.

3

Permits Screen

ActivePermitCard (reused), VehicleCard list (plate + make/model + delete, guest variant with dashed border), PricingCard grid (Semester/Monthly/Daily with shine buttons). Entrance animations. Tab 3 functional.

4

Map Screen

MapCanvas with campus image overlay. SearchBar (LiquidGlass), FilterChipRow (EV/ADA/Faculty/Visitor). LotPin components with pulse-ring animations. LotDetailSheet (slide-up, dynamic color per lot status). Tab 2 functional.

5

Access Gate Screen + Auth

ScannerCore (concentric pulse rings, scan laser, NFC icon). AuthorizedVehicleCard. HUD grid background. AuthGate with AppAuth OIDC PKCE. Token storage in Android Keystore. Tab 4 functional. Full navigation flow complete.

6

Real Data Layer + Polish

Wire Retrofit/Moshi client. Room persistence. WebSocket client for live occupancy. DataStore preferences. Replace fake ViewModels with real repositories. Offline queue for scans. Error states, loading skeletons, empty states. ktlint + detekt. Unit tests for domain/data layers.
