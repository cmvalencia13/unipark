Key decisions

Auth Strategy

NextAuth.js with Keycloak provider. JWT from Keycloak forwarded as Bearer token to Spring Boot API. Role claims validated on both sides.

API Communication

Next.js Server Components call Spring Boot directly (server-to-server). Client mutations use fetch with JWT from session. No BFF duplication.

Role Gating

Next.js middleware checks role claim from session. Admin pages visible to admin+. Superadmin pages visible only to superadmin. Backend @PreAuthorize as defense-in-depth.

Design System

Same "cyber-academic" dark theme as Android app. Tailwind config with Unipark tokens. Glassmorphism panels, Sora headings, Material Symbols icons.
