Section 2 — Admin API Contract

New Spring Boot endpoints to support the admin dashboard

User Management superadmin

GET /v1/admin/users List users (paginated, filterable by role, status, search)
GET /v1/admin/users/{id} User detail with vehicles, violations, scan history
PATCH /v1/admin/users/{id} Update role, active flag, driver category

Dashboard Stats admin+

GET /v1/admin/stats Occupancy per lot, violation counts, user counts, today's scans

Violations Console admin+

GET /v1/admin/violations All violations (paginated, filterable by status, lot, date)
PATCH /v1/admin/violations/{id} Approve or dismiss with resolution note

Audit & Settings superadmin

GET /v1/admin/audit Paginated audit log with date/actor/action filters
GET /v1/admin/settings Global thresholds, rate limits, system config
PATCH /v1/admin/settings Update global configuration

Spring Security setup for these endpoints

// SecurityConfig.kt — method-level security
@PreAuthorize("hasRole('ADMIN')")
fun getStats() { ... }
@PreAuthorize("hasRole('SUPERADMIN')")
fun listUsers() { ... }
// All /v1/admin/** requires authentication + admin role
.requestMatchers("/v1/admin/**").hasAnyRole("ADMIN", "SUPERADMIN")
